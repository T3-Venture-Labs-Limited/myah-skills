# Catalog Schema

This document defines the schema for the Myah skill marketplace catalog.
It covers the SKILL.md frontmatter format, the bundle.yaml manifest format,
and the generated catalog.json structure.

All schemas are versioned together. The current version is **1**.

## SKILL.md Frontmatter

Every skill is a YAML-frontmatter Markdown file. The frontmatter is a superset
of the Hermes-native schema with a `marketplace:` block for marketplace-specific
metadata.

### HermesFrontmatter

Fields recognized by the Hermes agent runtime.

| Field | Type | Required | Constraints | Description |
|-------|------|----------|-------------|-------------|
| `name` | `string` | Yes | `^[a-z][a-z0-9-]{1,63}$` | Unique skill identifier, kebab-case. |
| `description` | `string` | Yes | 20–500 characters | One-line description of what the skill does. |
| `license` | `string` | Yes | SPDX identifier or custom string | License under which the skill is distributed. |
| `role` | `string` | Yes | One of: `tool`, `router`, `workflow` | Hermes runtime role classification. |
| `version` | `string` | Yes | Semver (e.g. `1.0.0`) | Informational version; pinning uses commit SHA. |

### MarketplaceFrontmatter

Fields inside the `marketplace:` block. Purely additive — Hermes ignores unknown keys.

| Field | Type | Required | Constraints | Description |
|-------|------|----------|-------------|-------------|
| `category` | `string` | Yes | One of the curated category list | Broad category driving category filter. |
| `tags` | `string[]` | Yes | 1–10 items, lowercase-kebab | Specific tags driving search. |
| `personas` | `PersonaAxis` | Yes | All 6 keys present, 0–100 | Persona radar scores. |
| `summary` | `string` | Yes | 20–120 characters | One-line marketplace summary. |
| `featured` | `boolean` | No | Defaults to `false` | If `true`, shown in the Featured section. |
| `requires` | `object` | No | See below | Declarative capability list for the approval screen. |
| `requires.tools` | `string[]` | No | — | Tool names the skill requires. |
| `requires.mcp` | `string[]` | No | — | MCP server names the skill requires. |
| `requires.env` | `string[]` | No | — | Environment variable names the skill expects. |
| `capabilities` | `object[]` | No | See below | Human-readable capability descriptions for the approval screen. |
| `capabilities[].title` | `string` | Yes | 1–80 characters | Short title of the capability. |
| `capabilities[].description` | `string` | Yes | 1–200 characters | Description of what the skill can do. |
| `icon` | `string` | No | Valid URL | Custom icon URL. Falls back to deterministic SkillGlyph if absent. |
| `author` | `object` | Yes | — | Attribution for the skill. |
| `author.name` | `string` | Yes | — | Author or team name. |
| `author.url` | `string` | No | Valid URL | Link to author website or profile. |

### SkillFrontmatter

The complete frontmatter object — Hermes fields plus the marketplace block.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| *(Hermes fields)* | `HermesFrontmatter` | Yes | All Hermes fields at the top level. |
| `marketplace` | `MarketplaceFrontmatter` | Yes | Marketplace-specific metadata block. |

## Bundle Manifest

Bundles are defined by a `bundle.yaml` file inside `bundles/<bundle-slug>/`.

### BundleManifest

| Field | Type | Required | Constraints | Description |
|-------|------|----------|-------------|-------------|
| `slug` | `string` | Yes | `^[a-z][a-z0-9-]{1,63}$` | Unique bundle identifier, kebab-case. |
| `name` | `string` | Yes | 1–120 characters | Human-readable bundle name. |
| `summary` | `string` | Yes | 20–500 characters | One-line description of the bundle. |
| `category` | `string` | Yes | One of the curated category list | Broad category for filtering. |
| `skills` | `string[]` | Yes | 1–50 items, each a valid skill `name` | Member skills included in the bundle. |
| `personas` | `PersonaAxis` | Yes | All 6 keys present, 0–100 | Bundle-level persona radar scores. |
| `featured` | `boolean` | No | Defaults to `false` | If `true`, shown in the Featured section. |
| `author` | `object` | Yes | — | Attribution for the bundle. |
| `author.name` | `string` | Yes | — | Author or team name. |
| `author.url` | `string` | No | Valid URL | Link to author website or profile. |

## Generated Catalog

The `catalog.json` file is generated by `scripts/build-catalog.ts` on every PR merge.
It is a single flat file containing every skill and bundle, plus security audit results.

### CatalogEntry

A single skill entry in the catalog.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `path` | `string` | Yes | Relative path to the skill directory (e.g. `skills/shopify-order-analyzer`). |
| `frontmatter` | `SkillFrontmatter` | Yes | Full parsed frontmatter object. |
| `description_html` | `string` | Yes | Rendered HTML of the frontmatter `description` field. |
| `body_html` | `string` | Yes | Rendered HTML of the SKILL.md body (below frontmatter). |
| `size_bytes` | `number` | Yes | Total size in bytes of all files in the skill directory. |
| `files` | `string[]` | Yes | List of relative file paths inside the skill directory. |
| `security` | `SecurityAuditResult` | Yes | Results from the three CI security checks. |
| `source` | `'internal' \| 'skills.sh'` | No | Where the skill originates. Defaults to `'internal'` if absent. |
| `identifier` | `string` | No | Unique identifier on the source platform. For `skills.sh` entries, the skills.sh slug. |
| `external_audit` | `ExternalAudit \| null` | No | Audit results from the external source. Only present for `source: 'skills.sh'` entries. |

### ExternalAudit

Audit results fetched from the external source (skills.sh).
Only present for entries with `source: 'skills.sh'`.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `trust_hub` | `AuditCheck` | Yes | Gen Agent Trust Hub scan result. |
| `socket` | `AuditCheck` | Yes | Socket.dev supply-chain scan result. |
| `snyk` | `AuditCheck` | Yes | Snyk CVE + license scan result. |
| `last_scanned` | `string` | Yes | ISO 8601 timestamp of the most recent scan. |
| `detail_url` | `string` | Yes | URL to the full audit report on the external platform. |

### AuditCheck

A single audit check from an external source.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `status` | `'pass' \| 'warn' \| 'fail'` | Yes | Result of the check. Entries with any `fail` are excluded from the catalog. |
| `detail_url` | `string` | No | URL to the detailed report for this specific check. |

### Catalog

The root object of `catalog.json`.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `version` | `number` | Yes | Schema version (currently `1`). |
| `generated_at` | `string` | Yes | ISO 8601 timestamp of generation. |
| `commit_sha` | `string` | Yes | Full 40-character Git commit SHA at generation time. |
| `skills` | `Record<string, CatalogEntry>` | Yes | Map of skill name → catalog entry. |
| `bundles` | `Record<string, BundleManifest>` | Yes | Map of bundle slug → bundle manifest. |
| `categories` | `string[]` | Yes | Sorted list of all unique categories across skills and bundles. |

## Security Audit

Every skill is scanned by three CI checks. Results are stored per-skill in `catalog.json`.

### SecurityAuditResult

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `genAgentTrustHub` | `object` | Yes | Gen Agent Trust Hub scan result. |
| `genAgentTrustHub.status` | `string` | Yes | One of: `pass`, `warning`, `fail`, `high_risk`. |
| `genAgentTrustHub.url` | `string` | Yes | URL to the full report on the tool's site. |
| `genAgentTrustHub.checked_at` | `string` | Yes | ISO 8601 timestamp of the check. |
| `socket` | `object` | Yes | Socket.dev supply-chain scan result. |
| `socket.status` | `string` | Yes | One of: `pass`, `alerts`. |
| `socket.alerts` | `number` | Yes | Number of alerts found (0 = clean). |
| `socket.url` | `string` | Yes | URL to the full report on the tool's site. |
| `snyk` | `object` | Yes | Snyk CVE + license scan result. |
| `snyk.status` | `string` | Yes | One of: `safe`, `low_risk`, `med_risk`, `high_risk`, `critical`. |
| `snyk.url` | `string` | Yes | URL to the full report on the tool's site. |

## Persona Axis

The persona radar chart uses six axes. Every skill and bundle must declare scores
for all six.

### PersonaAxis

| Field | Type | Required | Constraints | Description |
|-------|------|----------|-------------|-------------|
| `developer` | `number` | Yes | 0–100 integer | Building, coding, API work. |
| `researcher` | `number` | Yes | 0–100 integer | Deep investigation, literature review. |
| `analyst` | `number` | Yes | 0–100 integer | Data analysis, metrics, reporting. |
| `operator` | `number` | Yes | 0–100 integer | Day-to-day operations, workflows. |
| `creator` | `number` | Yes | 0–100 integer | Content creation, design, writing. |
| `support` | `number` | Yes | 0–100 integer | Customer support, troubleshooting. |

## Full Examples

### SKILL.md Example

```yaml
---
name: shopify-order-analyzer
description: Analyze Shopify order data, surface anomalies, draft customer emails.
license: MIT
role: tool
version: 1.0.0

marketplace:
  category: e-commerce
  tags:
    - shopify
    - orders
    - analytics
  personas:
    developer: 40
    researcher: 30
    analyst: 85
    operator: 75
    creator: 20
    support: 60
  summary: One-line marketplace summary (<= 100 chars).
  featured: false
  requires:
    tools: [web-search, file-read]
    mcp: []
    env: []
  author:
    name: Myah Team
    url: https://myah.dev
---

# Shopify Order Analyzer

Analyze your Shopify store's order data to identify trends, anomalies,
and opportunities. This skill can:

- Summarize daily/weekly order volumes
- Flag unusual refund rates
- Draft personalized customer follow-up emails
- Surface best-selling products by region
```

## External Source Directory

External skills (sourced from skills.sh or other registries) are placed in the
`external/<slug>/` directory. Each external skill directory must contain a
`skill.yaml` file with the skill's frontmatter and audit data.

### External skill.yaml

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `frontmatter` | `SkillFrontmatter` | Yes | Full frontmatter object (same schema as internal SKILL.md). |
| `external_audit` | `ExternalAudit` | Yes | Audit results from the external source. |

The build script reads every `external/<slug>/skill.yaml`, merges it into the
catalog, and **skips** any entry where any audit check has `status: 'fail'`.
Skipped entries emit a `console.warn` during the build.

Internal skills (under `skills/`) always have `source: 'internal'` and no
`external_audit` field. External skills always have `source: 'skills.sh'`,
an `identifier` matching their slug, and placeholder values for `body_html`,
`files`, and `size_bytes`.

## Curated Category List (v1)

The following categories are valid for `marketplace.category` and `bundle.category`:

- `e-commerce`
- `finance`
- `productivity`
- `analytics`
- `communication`
- `development`
- `marketing`
- `operations`
- `research`
- `support`

New categories may be added by updating this list and regenerating the catalog.
