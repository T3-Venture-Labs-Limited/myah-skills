# Myah Skills

The Myah skill marketplace — a curated collection of Hermes agent skills. Each skill is a
SKILL.md file that defines capabilities, triggers, and behaviors for the Myah
agent runtime.

## Repository structure

```
myah-skills/
├── README.md
├── LICENSE
├── SECURITY.md
├── CATALOG_SCHEMA.md
├── catalog.json           # auto-generated by CI — do not edit by hand
├── scripts/
│   ├── build-catalog.ts
│   └── validate-skill.ts
├── skills/
│   └── <skill-name>/
│       └── SKILL.md
├── bundles/
│   └── <bundle-slug>/
│       ├── bundle.yaml
│       └── README.md
└── external/              # skills sourced from external registries
    └── <slug>/
        └── skill.yaml
```

## Adding a skill

1. Create a directory under `skills/<skill-name>/`
2. Add a `SKILL.md` file following the [CATALOG_SCHEMA.md](CATALOG_SCHEMA.md)
3. Run `npm run validate` — catches schema errors before you push
4. Submit a PR — CI rebuilds `catalog.json` automatically on merge

## Updating a skill

1. Edit the relevant `skills/<skill-name>/SKILL.md`
2. Run `npm run validate` — catches broken bundle references and schema errors
3. Submit a PR — CI rebuilds `catalog.json` automatically on merge

> `catalog.json` is regenerated by CI on every push to `main`. You do not need to run
> `npm run build-catalog` locally or commit `catalog.json` manually.

## Adding a bundle

1. Create a directory under `bundles/<bundle-slug>/`
2. Add `bundle.yaml` and `README.md` (see [CATALOG_SCHEMA.md](CATALOG_SCHEMA.md) for the full schema)
3. Run `npm run validate` — confirms all referenced skills exist
4. Submit a PR — CI handles the rest

## Quick examples

### Minimal SKILL.md

```markdown
---
name: my-skill
description: "One-line description of what this skill does (20–500 chars)"
license: MIT
role: workflow
version: 1.0.0
marketplace:
  category: productivity
  tags:
    - my-tag
  personas:
    developer: 60
    researcher: 40
    analyst: 50
    operator: 70
    creator: 30
    support: 45
  summary: "Short marketplace summary (20–120 chars)"
  featured: false
  author:
    name: Your Name
    url: https://example.com
---

# My Skill

Describe what the skill does and how to use it.
```

### Minimal bundle.yaml

```yaml
slug: my-bundle
name: My Bundle
summary: A short description of what this bundle does (20–500 chars)
category: productivity
skills:
  - my-skill
  - another-skill
personas:
  developer: 60
  researcher: 40
  analyst: 50
  operator: 70
  creator: 30
  support: 45
featured: false
author:
  name: Your Name
  url: https://example.com
```

See [CATALOG_SCHEMA.md](CATALOG_SCHEMA.md) for the full schema, all valid categories, and
a complete catalog.json example.

## Development

```bash
npm install

# Validate all skills against the schema
npm run validate

# Rebuild catalog.json from current skill files (normally handled by CI)
npm run build-catalog

# Run tests
npm test
```
