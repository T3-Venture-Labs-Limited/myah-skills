## Adding a skill

1. Create a directory under `skills/<skill-name>/`
2. Add a `SKILL.md` file following the [CATALOG_SCHEMA.md](CATALOG_SCHEMA.md)
3. Submit a PR вЂ” CI validates the schema and rebuilds `catalog.json` automatically on merge

## Updating a skill

1. Edit the relevant `skills/<skill-name>/SKILL.md`
2. Submit a PR вЂ” CI rebuilds `catalog.json` automatically on merge

> `catalog.json` is regenerated by CI on every push to `main`. You do not need to run
> `npm run build-catalog` locally or commit `catalog.json` manually.

## Adding a bundle

1. Create a directory under `bundles/<bundle-slug>/`
2. Add `bundle.yaml` and `README.md` (see [CATALOG_SCHEMA.md](CATALOG_SCHEMA.md) for the full schema)
3. Submit a PR вЂ” CI handles the rest

## Development

```bash
npm install

# Validate all skills against the schema
npm run validate

# Rebuild catalog.json from current skill files (normally handled by CI)
npm run build-catalog

# Run tests
npm test
```
